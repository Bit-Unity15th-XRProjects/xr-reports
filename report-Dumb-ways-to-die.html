1. 미구현 핵심 시스템 착수: StageController, ScoreManager
예상 시스템 중 2개가 감지되지 않았습니다. 핵심 시스템이 없으면 게임 루프를 완성할 수 없습니다.
  1. 우선순위 1위: StageController — 게임 플로우의 핵심이므로 먼저 구현
  2. 최소 기능부터 시작: 빈 클래스 → public 메서드 시그니처 → 기본 로직 → 테스트

  4. MonoBehaviour가 아닌 일반 C# 클래스로 로직을 분리하면 테스트가 쉬움
2. 브랜치 정리 및 main 통합

  2. git log --oneline --graph --all 로 브랜치 구조 시각화
  3. 가장 변경이 적은 브랜치부터 main에 병합
  4. 충돌 발생 시 팀원과 함께 resolve — 혼자 임의로 해결하지 않기
