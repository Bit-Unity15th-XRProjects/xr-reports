프로젝트 현황 요약

이 팀은 AR 위치기반 몬스터 RPG를 개발 중이며, GPS 기반 필드 맵에서 몬스터를 만나 전투하는 게임 루프의 골격을 완성했다. BattleManager의 HP/XP 스케일링, MonsterSpawner의 레벨 기반 출현 확률, InventoryController의 아이템 장착 시스템, LevelManager의 경험치 체계 등 핵심 시스템이 실제 동작하는 수준으로 구현되어 있다. 그러나 GpsMapPlayerMover의 초기화 버그(_isInitialized 미설정)와 DefeatController의 NPE 버그가 수정되지 않으면 실제 디바이스 시연에서 크래시할 위험이 크다. SaveManager와 SoundManager가 완전히 부재하고, LimChaesung 브랜치의 VFX 작업이 미머지된 채 11일간 개발이 정체되어 있어, 마감까지 버그 수정과 미머지 브랜치 통합, 최소 저장/사운드 구현에 집중해야 시연 가능한 프로토타입을 완성할 수 있다.

코드 리뷰

BattleManager.cs개선필요
403줄로 권장 300줄 초과. GetBaseMaxHpByGrade(), GetScaledMaxHp(), GetXpRewardByGrade() 등 등급별 HP/XP 스케일링 공식이 정확하게 구현됨. 타이머 시스템(battleTimerText, battleTimeRadialImage)과 무기 드롭 확률(weaponDropChanceNormal=30%, Rare=15%, Epic=5%)도 완비. 그러나 GetRewardItem()이 null만 반환하는 스텁이고, CreateRandomItem()에서 ScriptableObject.CreateInstance<Item>()으로 런타임 에셋을 생성하는데 이는 메모리 릭 가능성이 있음. Singleton이 null 체크 없이 Instance = this로만 할당됨.
MonsterSpawner.cs개선필요
레벨 기반 스폰 확률 시스템(UpdateProbabilityValues)이 구현되어 있고, DecideGradeByLevel()로 Normal/Rare/Epic 확률 분배가 동작함. EnterBattle()에서 씬 전환과 카메라 교체(fieldCamera↔arCamera) 로직이 있어 게임 루프의 핵심 연결점 역할. HideInGameSceneRootsExcept()로 배틀 진입 시 필드 오브젝트를 비활성화하는 처리도 있음. 그러나 static 변수(FieldPaused, FieldCameraRef, ArCameraRef, _hiddenInGameSceneRoots)를 과도하게 사용해 전역 상태 의존도가 높고, battleSceneName이 빈 문자열일 때만 경고하고 반환하는 방어 코드는 있으나 씬 존재 여부 검증은 없음.
GpsMapPlayerMover.cs심각
GPS 좌표를 월드 좌표로 변환하는 ConvertGpsToWorld()와 GetMeterOffset()의 수학 공식은 정확함. 에디터 모킹(UpdateEditorMock), 터치 드래그(HandleTouchDrag), 애니메이터 연동 등 기능이 풍부함. 그러나 치명적 문제: _isInitialized 플래그가 Start() 코루틴 완료 후에도 true로 설정되지 않아 Update()가 항상 즉시 return하여 실제 디바이스에서 플레이어가 움직이지 않을 가능성이 높음. 341줄로 책임이 과다(GPS, 애니메이션, UI, 터치 입력)하며 분리가 필요함.
Monster.cs양호
MonsterGrade/MonsterType enum과 TypeSpriteScales struct로 등급×타입 조합을 체계적으로 관리. Init()에서 spawner/grade/type을 설정하고 ApplyVisualByGradeType()에서 Resources 폴더의 스프라이트를 로딩하여 적용하는 파이프라인이 완성됨. 배회 AI(wanderSpeed, wanderRetargetInterval)와 도주 시스템(fleeAfterSeconds, fleeSpeed)이 Inspector에서 조정 가능. GetDisplayName()으로 '[등급] 타입' 형식 이름 생성. 331줄로 다소 길지만 기능 대비 적절한 수준.
InventoryController.cs개선필요
AddItem(), ToggleEquip(), SelectSlot(), RefreshSlots() 등 인벤토리 핵심 기능이 모두 구현됨. Singleton에서 null 체크와 additive 씬 지원이 있어 다른 Singleton보다 견고함. 슬롯 정렬(GetSlotNumber)과 장착 무기 추적(EquippedWeapon 프로퍼티)도 동작함. 그러나 변수명 오타 'OutPanul'(OutPanel이어야 함), InventorySlotUI.DestoryItemData() 메서드명 오타, onUseItem() PascalCase 미준수가 코드 품질을 낮춤.
DefeatController.cs심각
치명적 버그: ShowDefeat() 또는 Update()에서 'DefeatUI == null && DefeatUI.activeInHierarchy' 조건이 사용되는데, DefeatUI가 null이면 두 번째 조건에서 NullReferenceException이 발생함. 올바른 조건은 'DefeatUI != null && DefeatUI.activeSelf'여야 함. OnClickToMap()의 복귀 로직이 BattleManager.ReturnToFieldWithoutReward()와 중복되어 있고, 하드코딩된 씬 이름 'BattleScene'이 MonsterSpawner의 battleSceneName과 불일치할 위험이 있음.
BattleSelectUI.cs개선필요
몬스터 리스트 UI를 동적 생성하고 토글 애니메이션(listSlidePanel 스르륵 열기)을 구현. 등급별 텍스트 색상(colorGradeNormal/Rare/Epic)과 한국어 폰트 폴백(GetKoreanFont)도 처리됨. OnStartBattle()에서 선택된 몬스터를 BattleSelectionContext에 저장하고 MonsterSpawner.EnterBattle()을 호출하는 통합 흐름이 완성됨. 399줄로 과도하게 길며 UI 생성 로직과 애니메이션 로직의 분리가 필요함.
MapGridGenerator.cs개선필요
GPS 좌표를 Web Mercator 타일 좌표로 변환하여 3×3 그리드를 생성하고 MapTileRenderer로 OSM 타일을 다운로드하는 핵심 맵 시스템. GetTileCoord(), GetSubTilePos()의 좌표 변환 수학이 정확함. 그러나 에이전트 리포트에 따르면 GPS 폴백 좌표에 오타(7.494920f → 37.494920f여야 함)가 있어 GPS 실패 시 잘못된 위치에 맵이 생성될 수 있음. 줌 레벨 15, 그리드 크기 3×3, 타일 간격 10유닛이 모두 하드코딩됨.

아키텍처 평가

Singleton 패턴을 BattleManager, LevelManager, InventoryController, VictoryController, DefeatController에 광범위하게 사용하나, 대부분 null 체크나 DontDestroyOnLoad 없이 'Instance = this' 단순 할당만 수행. BattleSelectionContext는 static 클래스로 씬 간 데이터 전달에 사용되며 이는 학생 프로젝트에서 합리적인 선택. 씬 기반 구조(InGame→BattleScene)로 필드와 전투를 분리했고, SceneManager.LoadSceneAsync(Additive)로 씬을 중첩 로드하는 방식이 적절함. 네임스페이스를 ARRPG.Level, ARRPG.Inventory 등으로 분리한 것은 좋으나 GamePlay.Data, Core.Location, ARRPG.MAP 등 불일치가 아쉬움. MonsterSpawner의 static 변수(FieldPaused, FieldCameraRef)를 통한 전역 상태 관리가 클래스 간 암묵적 결합을 만들어 디버깅을 어렵게 함.

심각도파일설명
CRITICALDefeatController.csnull 체크 논리 오류: 'DefeatUI == null && DefeatUI.activeInHierarchy' 조건에서 DefeatUI가 null일 때 두 번째 피연산자 접근 시 NullReferenceException 발생. '!= null'로 수정하고 activeInHierarchy 대신 activeSelf 사용 필요
CRITICALGpsMapPlayerMover.csStart() 코루틴 완료 후 _isInitialized가 true로 설정되지 않아 Update()의 'if (!_isInitialized || !_isReady) return;' 조건에 항상 걸려서 실제 디바이스에서 플레이어 이동이 동작하지 않을 가능성이 높음
HIGHSimpleFollowCamera.csAwake()에서 '_currentDistance = Mathf.Clamp(_minDistance, _minDistance, _maxDistance)' 호출 — 첫 번째 인자가 _minDistance가 아닌 _offset.magnitude여야 함. 현재 코드는 항상 _minDistance를 반환하여 줌 초기값이 최소값으로 고정됨
HIGHSpawnProbUI.csUpdate()에서 매 프레임 FindObjectOfType<MonsterSpawner>() 호출 — 60fps 기준 초당 60회 리플렉션 기반 검색으로 심각한 성능 저하 유발. Start()에서 캐싱하거나 Singleton 패턴으로 접근해야 함
HIGHMapGridGenerator.csGPS 폴백 좌표 오타: 위도값이 7.494920f로 되어 있어 GPS 초기화 실패 시 아프리카 해안(적도 근처)에 맵이 생성됨. 37.494920f(서울)로 수정 필요
MEDIUMBattleManager.csGetRewardItem() 메서드가 항상 null을 반환하는 스텁 상태. 승리 시 아이템 보상 로직이 CreateRandomItem()으로 별도 구현되어 있어 GetRewardItem()은 데드 코드이지만, 다른 곳에서 호출 시 보상 누락 발생
MEDIUMBattleUI.csStart()에서 FindObjectOfType<BattleManager>() 호출 후 null 체크 없이 Update()에서 battleManager의 프로퍼티에 접근. BattleManager가 아직 생성되지 않은 타이밍에 NPE 발생 가능
MEDIUMBattleManager.csCreateRandomItem()에서 ScriptableObject.CreateInstance<Item>()으로 런타임에 에셋을 생성하는데, 이 인스턴스가 씬 전환 시 해제되지 않으면 메모리 릭 발생. Destroy() 호출이나 인벤토리 관리와 연계 필요
MEDIUMMonsterSpawner.csstatic 변수 FieldPaused가 배틀 진입/복귀 시 수동으로 토글되는데, 예외 발생 시 true 상태로 남아 필드 몬스터가 영구 정지될 수 있음. try-finally나 상태 머신으로 보호 필요
LOWInventorySlotUI.cs메서드명 오타 DestoryItemData() — DestroyItemData()여야 함. 외부에서 이 메서드를 호출하는 코드가 오타된 이름으로 연결되어 있어 리팩토링 시 혼란 유발
