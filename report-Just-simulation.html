프로젝트 현황 요약

이 팀은 OpenXR 기반 VR 파괴 시뮬레이터 'Just Simulation'을 개발 중이며, Unity 6000.3.8f1에 XR Interaction Toolkit, XR Hands 등 적절한 VR 패키지를 세팅한 상태. 그러나 현재 가장 심각한 문제는 40개 커밋 분량의 모든 게임 코드가 3개의 미병합 브랜치에만 존재하고, main 브랜치는 빈 VR 템플릿 상태라는 점. verpivia-1이 UI시스템과 AudioManager를, 김연준이 맵/스테이지를 각각 독립적으로 개발하고 있으나 한 번도 통합된 적이 없어, 기대되는 10개 핵심 시스템(GameManager, BatSwing, FractureSystem 등) 중 main에서 확인 가능한 것은 0개. 가장 시급한 것은 이번 주 내로 브랜치를 main에 병합하여 통합 빌드를 만들고, 핵심 게임플레이 루프(무기 → 파괴 → 점수)가 실제로 동작하는지 검증하는 것이며, .csproj 파일의 .gitignore 추가와 커밋 메시지 컨벤션 도입도 병행해야 함.

권고사항 (Top 3)

1. 긴급 브랜치 통합 및 main 안정화
현재 60커밋 이상의 코드가 main에 병합되지 않은 상태로, 시간이 지날수록 병합 난이도가 기하급수적으로 증가함. 프로젝트 데드라인이 다가올수록 이 기술 부채는 치명적이 됨.
  1. 팀 전체가 모여 각 브랜치의 현재 상태와 의존성을 파악
  2. Test/Stage-Design(22 ahead)을 먼저 main에 병합 (맵/레벨 기반)
  3. Test/New-UISystem-Design(36 ahead)을 그 위에 병합
  4. Test/UI_Archetectures(2 ahead)는 UI 리팩토링이므로 마지막에 병합
  5. 병합 후 전체 빌드 및 VR 기기 테스트 실행
2. .gitignore 정리 및 프로젝트 위생 개선
자동 생성 파일(.csproj, .sln)이 버전 관리에 포함되어 Hot Files 상위를 독점하고 있음. 이는 불필요한 병합 충돌과 저장소 비대화의 원인.
  1. .gitignore에 *.csproj, *.sln, *.suo, .vs/ 추가
  2. git rm --cached로 기존 추적 파일 제거
  3. Unity 공식 .gitignore 템플릿(github/gitignore)과 비교하여 누락 항목 추가
3. 핵심 게임플레이 루프 우선 구현
VR 파괴 시뮬레이터의 핵심인 '물건을 잡아서 휘두르고 부수는' 루프가 main에서 확인 불가. UI 리팩토링(3개 브랜치 중 2개)에 비해 핵심 게임플레이 구현 비중이 낮아 보임.
  1. BatSwing/Weapon → FractureSystem → ScoreManager 순서로 핵심 루프 구현
  2. XR Grab Interactable 기반 무기 인터랙션 프로토타입 제작
  3. Fracture 시스템은 RayFire나 자체 메시 분할 중 방향 확정
  4. 최소 1개 스테이지에서 엔드투엔드 플레이 가능하도록 통합

Assets/Scenes/BasicScene.unity개선필요

Assembly-CSharp.csproj개선필요
95KB 크기의 프로젝트 파일이지만 커스텀 스크립트 참조가 없음. Hot Files에서 .csproj 파일들이 13회씩 변경된 것은 Unity 에디터가 패키지 변경 시 자동 재생성하기 때문. 이 파일들은 .gitignore에 추가하는 것이 일반적 관행.
